chapter 2. 리스트와 딕셔너리

딕셔너리

  • 리스트를 보완
  • 타입 이름 dict
  • 검색에 사용할 키와 키에 연관된 값을 저장
  • 해시 테이블, 연관배열
  • 분할상환 복잡도로 상수 시간에 원소를 삽입하고 찾을 수 있음
  • 동적인 정보를 관리하는데 딕셔너리가 가장 이상적

기본 딕셔너리 모습

{Key1:Value1, Key2:Value2, Key3:Value3, …}

Better way 11. 시퀀스를 슬라이싱하는 방법을 익혀라.

  • 파이썬에는 시퀀스를 여러 조각(슬라이스 slice)로 나누는 슬라이싱 구문 있음
  • 슬라이싱으로 최소한의 노력으로 시퀀스에 들어 있는 아이템의 부분집합에 쉽게 접근
  • 슬라이싱 기본 구문 : 리스트[시작:끝]
  • 시작 인덱스에 있는 원소는 슬라이스에 포함
  • 끝 인덱스에 있는 원소는 불포함
  • 리스트 맨 앞부터 슬라이싱 할때는 0 생략
  • 리스트 끝까지 슬라이싱 할때는 끝 인덱스 생략
  • 리스트의 끝에서부터 원소를 찾고싶을때는 음수 인덱스 사용
  • 슬라이싱 할때 리스트의 인덱스 범위를 넘어가는 시작과 끝 인덱스는 무시
  • 인덱스 직접 접근 시 예외 발생
  • a[-0:] 일 경우 원래의 리스트를 복사한 리스트 얻음
  • 슬라이싱한 결과로 얻은 리스트를 변경해도 원래 리스트는 바뀌지 않음
  • 대입에 슬라이스 사용시 원본 리스트에 지정한 범위에 들어있는 원소 변경
  • 언패킹 대입(a,b = c[:2])과 달리 슬라이스 대입에서는 슬라이스와 대입되는 리스트의 길이가 같을 필요 없음
  • 시작과 끝 인덱스를 모두 생략하면 원래 리스트를 복사한 새 리스트를 얻음

is 와 == 의 차이

  • is는 ==과 달리 값을 비교하는게 아니라 레퍼런스, 즉 포인터를 비교함
  • python 에서는 변수는 내부적으로 데이터를 가르키는게 아니라 인스턴스 포인터를 가르킴

https://ponyozzang.tistory.com/292

  • 시작과 끝 인덱스가 없는 슬라이스에 대입하면 새 리스트 생성 대신 복사본으로 덮어씀

슬라이싱 할때는 간결하게 하라. 시작 인덱스에 0을 넣거나, 끝 인덱스에 시퀀스 길이를 넣지 말라.
슬라이싱은 범위를 넘어가는 시작 인덱스나 끝 인덱스도 허용한다.
리스트 슬라이스에 대입하면 원래 시퀀스에서 슬라이스가 가리키는 부분을 대입 연산자 오른쪽에 있는 시퀀스로 대치한다.


Better way 12. 스트라이드와 슬라이스를 한 식에 함께 사용하지 말라.

스트라이드 (stride)

  • 리스트[시작:끝:증가값]으로 일정한 간격을 두고 슬라이싱 할 수 있는 특별한 구문 제공
  • 역으로 뒤짚기
  • 슬라이싱 구문에 스트라이딩 까지 들어가면 아주 혼란스러워짐
  • 코드 밀도 높아져 읽기 어려움
  • 끝값을 증가값과 함께 사용하지 않아야함
  • 증가값을 사용할 경우 양수값으로 만들고 시작과 끝 인덱스를 생략
  • 시작이나 끝 인덱스와 함께 증가값을 사용해야 한다면 스트라이딩한 결과를 변수에 대입한 다음 슬라이싱 해야함

슬라이스의 시작, 끝, 증가값을 함께 지정하면 코드의 의미를 혼동하기 쉽다.
시작이나 끝 인덱스가 없는 슬라이스를 만들 때는 양수 증가값을 사용하라.
한 슬라이스 안에서 시작, 끝, 증가값을 함께 사용하지 말라.


Better way 13. 슬라이싱 보다는 나머지를 모두 잡아내는 언패킹을 사용하라.

  • 기본 언패킹의 한계점은 언패킹할 시퀀스의 길이를 미리 알고있어야 한다는 것

step 1) 인덱스와 슬라이싱

단점은 인덱스로 인한 오류(off-by-one error) : 1 차이나는 인덱스로 인한 오류

step 2) 별표식

  • 언패킹 패턴의 다른 부분에 들어가지 못하는 모든 값을 별이 붙은 부분에 다 담음
  • 짧고 읽기 쉽고 여러 줄 사이에 인덱스 경계 값이 어긋나서 오류가 발생할 여지 없음
  • 별표식만으로 언패킹할수 없음
  • 한 수준의 언패킹 패턴에 별표 식을 두개 이상 쓸 수 없음
  • 별표 식은 항상 list 인스턴스가 되며 언패킹하는 시퀀스에 남는 원소가 없으면 별표식은 빈 리스트가 됨

언패킹 대입에 별표 식을 사용하면 언패킹 패턴에서 대입되지 않는 모든 부분을 리스트에 잡아낼수 있다.
별표 식은 언패킹 패턴의 어떤 위치에든 놓을 수 있다.
리스트를 서로 겹치지 않게 여러 조각으로 나눌 경우, 슬라이싱과 인덱스를 사용하기 보다는 나머지 모두를 잡아내는 언패킹을 사용해야 실수할 여지가 훨씬 줄어든다.


Better way 14. 복잡한 기준을 사용해 정렬할 때는 key 파라미터를 사용하라.

  • list 내장타입에는 리스트의 원소를 여러 기준에 따라 정렬할 수 있는 sort 메서드 있음
  • 정렬에 사용하고 싶은 애트리뷰트가 객체에 들어 있는 경우 sort 의 key 파라미터를 이용해야함

참고 str, repr 차이

https://shoark7.github.io/programming/python/difference-between-repr-vs-str

  • 문자열 같은 기본타입의 경우 정렬하기 전 key 함수를 사용해 원소값을 변형할 수 있음
  • 여러 기준을 사용해 정렬하고 싶을 때 가장 쉬운 방법은 tuple 을 사용하는 것
  • 튜플은 기본적으로 비교 가능하며 자연스러운 순서가 정해져 있음
  • sort 에 필요한 lt 정의가 들어있음
  • 튜플의 각 위치를 이터레이션하면서 각 인덱스에 해당하는 원소를 한번에 하나씩 비교
  • 튜플이 반환하는 key 함수의 한 가지 제약사항은 모든 비교 기준의 정렬 순서가 같아야 한다는 점
  • 숫자 값의 경우 단항 부호 반전(-) 연산자를 통해 정렬 방향을 혼합
  • 모든 타입에 부호 반전을 사용할 수 없음
  • 리스트 타입의 sort 메서드는 key 함수가 반환하는 값이 서로 같은 경우 리스트에 들어 있던 원래 순서를 그대로 유지
  • key 함수를 사용해 tuple을 반환하게 하고 여기에 단항 부호 반전 연산을 활용하는 접근 방식이 코드가 더 적고 읽기도 쉬움
  • 꼭 필요할 때만 sort를 여러번 호출하는 방법을 사용하는 것을 권장

리스트 타입에 들어 있는 sort 메서드를 사용하면 원소 타입이 문자열, 정수, 튜플 등과 같은 내장 타입인 경우 자연스러운 순서로 리스트의 원소를 정렬할 수 있다.
원소 타입에 특별 메서드를 통해 자연스러운 순서가 정의돼 있지 않으면 sort 메서드를 쓸수 없다.
sort 메서드의 key 파라미터를 사용하려면 리스트의 각 원소 대신 비교에 사용할 객체를 반환하는 도우미 함수를 제공할 수 있다.
key 함수에서 튜플을 반환하면 여러 정렬 기준을 하나로 엮을 수 있다.
부호를 바꿀 수 없는 타입의 경우 여러 정렬 기준을 조합하려면 각 정렬 기준마다 reverse 값으로 정렬 순서를 지정하면서 sort 메서드를 여러 번 사용해야 한다.
